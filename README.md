# Продвинутый вектор
Учебный проект в Яндекс.Практикуме

## Описание
Аналог контейнера std::vector. Хранение элементов осуществляется последовательно во внутреннем массиве. Сам вектор хранит только указатель на этот массив данных и количество элементов в нём. При заполнении всего массива происходит перевыделение памяти для нового массива, размер которого в два раза больше предыдущего. При этом все данные копируюся/переносятся в новый массив.

## Технологии:
* C++
* STL
* RAII
* move semantics
* placement new operator
* динамическая память
* итераторы

## В векторе реализованы:
* отдельный RAII класс-обёртка над "сырой" неинициализированнлй памятью
* конструктор по умолчанию, конструкторы соответственно от одного и двух параметров
* конструтор копирования и оператор присваивания
* перемещающий конструктор и перемещающий оператор присваивания
* деструктор
* получение итераторов на начало и конец вектора
* добавление элемента в конец вектора PushBack
* добавление элемента в произвольную позицию вектора Insert
* конструирование элемента на месте и его добавление в конец вектора EmplaceBack
* конструирование элемента на месте и его добавление в произвольную позицию вектора Emplace
* удаление последнего элемента PopBack
* удаление элемента в произвольной позиции вектора Erase
* получение первого элемента вектора Front
* получение последнего элемента вектора Back
* получение произвольного элемента вектор с помощью оператора []
* метод Reserve для изменения размера массива памяти вектора
* метод Resize для изменения количества элементов вектора
* метод Size, который возвращает количесво элементов в списке
* метод Capacity, который возвращает ёмкость всего массива памяти, хранящегося в векторе
* метод Empty, который опредделяет пуст ли вектор
* метод Swap для обмена элементами между векторами
* метод очистки списка Clear, не бросающий исключений

## Будущие изменения:
* добавление конструктора от std::initializer_list
* добавление конструктора от пары итераторов
* добавление метода Insert пары итераторов

## Особенности продвинутого вектора:
* вставка элементов обеспечивает строгую гарантию безопасности
* при полном заполнении внутреннего массива вектора требуется перевыдление памяти и перенос всех элементов
* элементы в памяти хранятся последовательно из-за этого кэширование проходит легко
* перебор элементов в двух направлениях и доступ к произвольному элементу
* обращение к n-ному элементу осуществляется за время О(1)
* операция swap, перемещение вектора выполняются за константное время О(1) 
* вставка элемента в конец осуществляется за амортизированную O(1)
